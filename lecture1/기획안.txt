'my-community'라는 프로젝트를 생성해줘.

작업은 Todo 계획을 세워서 순차적으로 진행해줘.

요구사항:
1. React + Vite로 프로젝트 생성
2. Supabase MCP를 사용해서 데이터베이스 테이블 생성 및 연결
3. 백엔드 없이 Supabase를 직접 연결하여 작동하는 구조
4. 로컬에서 빌드 후 gitpages로 배포 (dist 폴더 배포)

개발 순서:
1) 프로젝트 초기 설정
2) Supabase 데이터베이스 설계 및 테이블 생성
3) 프론트엔드 개발 (기획안 참고)
4) Supabase 연동 구현
5)npm run build로 로컬 빌드
6) 빌드된 dist 폴더를 git pages로 배포하기. git actions 방식으로 배포하고, sleep으로 대기하다가 배포가 완료되면 유저에게 배포된 링크를 안내하기.


== 나만의 커뮤니티 기획서 ==

커뮤니티 주제
- 어떤 주제의 커뮤니티인가요? (예: 게임, 맛집, 반려동물, 스터디 등)
- 스터디

디자인 스타일
- 사이트 이름: hell스터디
- 색상 테마: 어두운 테마
- 분위기: 전문적인,

특별한 기능 아이디어
-  좋아요 기능

페이지별 특징(user flow를 고려)
1. 로그인 페이지
	- 아이디 입력란
	- 비밀번호 입력란
	- 로그인 버튼
	- 회원가입하러가기 버튼
	- 로그인 성공시 -> 게시물 목록 페이지로
2. 회원가입 페이지
	- 상단 네비게이션에 '뒤로가기버튼'
	- 아이디 입력란
	- 비밀번호 입력란
	- 회원가입 하기 버튼
	- 회원가입 성공 시 로그인 페이지로
3. 게시물 목록페이지
	- 상단 네비게이션 : '로그인한 아이디' 님 환영합니다. + 로그아웃 버튼
	- 메인콘텐츠 영역 : 게시물 목록(리스트 방식으로 한줄씩, 한페이지에 10개 표시 -> 현재 페이지 표시 + 게시물 목록 페이지네이션)
	- 게시물 목록 표시 요소 : 제목, 내용 미리보기 1줄, 작성자, 좋아요 수, 댓글수, 작성 시간
	- 게시물 클릭시 게시물 상세 페이지로
	- 우측 상단에 게시물 작성 버튼 -> 클릭시 게시물 작성 페이지로
4. 게시물 상세페이지
	- 상단 네비게이션에 '뒤로가기버튼'
	- 게시물 영역
		- 제목, 전체 내용, 작성자, 작성시간
		- 게시물 좋아요 버튼 + 좋아요 수
	- 댓글 영역
		- 전체 댓글 수
		- 댓글 입력란 + 등록버튼
		- 댓글 목록 영역(정렬기준: 최신순이 위로오게)
			- 댓글 : 작성자, 댓글 내용, 댓글 작성 시간
5. 게시물 작성 페이지
	- 상단 네비게이션에 '뒤로가기버튼'
	- 제목, 내용 입력란
	- 업로드 버튼

	- 내용
	- 작성자
	- 작성시간
	- 좋아요 수

	-indexId
	- 내용
	- 작성자
	- 작성시간
	- 연결된 게시물 indexId
